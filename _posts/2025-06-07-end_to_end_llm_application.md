## Project Introduction
With the rise of Large Language Models (LLMs), tasks like text summarization have become faster and more accessible than ever. In this project, I built a lightweight application that transforms long paragraphs into short, meaningful summaries using AWS Bedrock.

Beyond demonstrating the summarization capabilities of LLMs, this project highlights the design and implementation of an end-to-end system, integrating a simple frontend, a Flask-based backend, and cloud-based AI services.

## Prerequisite Installations
As we are using AWS Bedrock and developing an API, install the following python library:

<ol>
<li>Flask~=3.0.3</li>
<li>boto3~=1.40.55</li>
</ol>

Before invoking any model, ensure that the required foundation models are enabled and accessible in your AWS account. To verify this, navigate to Amazon Bedrock in the AWS Management Console and open the Chat/Text Playground. From there, select the desired model and test it using the chat interface to confirm it is functioning correctly.

## Major components of this project

<ol>
<li>Bedrock API: Used to access foundation models for summarization that handles prompt-based text summarization.</li>
<li>Boto3: Acts as the bridge between Flask backend and AWS services. It is used to invoke Bedrock runtime.</li>
<li>Flask (Backend + API Layer): Handles routing and request processing. It exposes endpoints for user interaction and connects frontend with AI backend.</li>
<li>Frontend: Simple HTML/CSS based UI using Jinja templates. It allows users to input long text and view summarized output.</li>

</ol>

## Bedrock model connector
The core logic of the application involves establishing a connection to the AWS Bedrock API using the Boto3 SDK and invoking a foundation model for summarization. In this implementation, the amazon.nova-micro-v1:0 model is used. A structured prompt is defined within user_message to clearly instruct the LLM on the summarization task.

The request payload consisting of the prompt, user input, model ID, and inference configuration is passed to the Bedrock converse API for model invocation. Once the model processes the request, the summarized output is extracted from the raw response and returned to the application.

```
import boto3
from botocore.exceptions import ClientError


class BedrockModel:
  def invokeModel(self, long_text: str = ""):
    #Create bedrock runtime Client
    client = boto3.client("bedrock-runtime", region_name = "us-east-1")

    #Set the model ID
    model_id = "amazon.nova-micro-v1:0"
    
    user_message = f"""
    # Role
    You are an expert content creator specializing in content summarization for generic tasks.

    # Task
    Your task is to understand the content from the input text, which is usually a long paragraph, and summarize this long content into short paragraphs by retaining all the meaningful information.

    # Input
    This is the input long content for summarization:
    {long_text}

    # Output
    Please return a summarized paragraph for the inputs in maximum of 2 to 3 sentences. It is forbidden to add any additional explanation, details or unnecessary structures.
    """

    conversation = [
      {
        "role": "user",
        "content":[{"text": user_message}], 
      }
    ]
      
    try:
    #Send message to the model
      bedrock_response = client.converse(
      modelId=model_id,
      messages=conversation,
      inferenceConfig={"maxTokens":50, "temperature":0.5, "topP":0.9}   
    )

    #Extract and print the response text
      response_text = bedrock_response["output"]["message"]["content"][0]["text"]
      return response_text

    except (ClientError, Exception) as e:
      print(f"ERROR: Can't invoke '{model_id}'. Reason: {e}")

      return str(e)
```

## Backend with Flask API
The backend of the application is built using Flask and acts as the bridge between the user interface and the AWS Bedrock service. It defines routes to handle user requests, process input data, and return the summarized output. When a user submits a paragraph through the frontend form, the Flask application captures the input, routes it to the appropriate endpoint, and invokes the Bedrock model through a dedicated service class. The response generated by the model is then returned to the client and rendered in the browser.

*app.py*

```
from flask import Flask, redirect, render_template, url_for, request
from model.model import BedrockModel

app = Flask(__name__)
bedrock_model = BedrockModel()

@app.route("/", methods=["POST", "GET"])
def home():
    if request.method == "POST":
        user = request.form["nm"]
        
        return redirect(url_for("user", usr=user))
    
    else:
        return render_template("data.html")

@app.route("/<usr>")
def user(usr):
    model_response = bedrock_model.invokeModel(usr)

    return f"<bady>{model_response}</body>"

if __name__ == "__main__":
    app.run(host='0.0.0.0', port=5000, debug=True)
```
